# Prompt Workbench 操作手册

这份手册面向第一次使用仓库的人，重点说明如何启动、打开和使用 `PromptWorkbench`。

## 1. 这个项目现在能做什么

当前仓库里已经补了一个本地可用的提示词工作台：

- 网页文件：[PromptWorkbench.html](/D:/study/提示词助手/langgpt-main/PromptWorkbench.html)
- 本地服务脚本：[server.js](/D:/study/提示词助手/langgpt-main/server.js)
- Windows 启动脚本：[run-server.cmd](/D:/study/提示词助手/langgpt-main/run-server.cmd)
- 日常模板库：[Prompts/日常AI对话模板.md](/D:/study/提示词助手/langgpt-main/Prompts/日常AI对话模板.md)

它适合日常和 AI 对话时快速生成可复制的提示词，不需要自己每次从零组织结构。

## 2. 启动方式

### 方式一：直接运行脚本

在仓库根目录运行：

```bat
run-server.cmd
```

默认会启动本地服务，端口是 `18888`。

### 方式二：手动运行 Node 服务

如果你想手动启动，也可以在根目录执行：

```powershell
node server.js
```

## 3. 打开地址

服务启动后，浏览器访问：

```text
http://localhost:18888/
```

如果 `localhost` 在当前环境不稳定，也可以访问：

```text
http://127.0.0.1:18888/
```

如果你想直接打开工作台页面，也可以访问：

```text
http://localhost:18888/PromptWorkbench.html
```

## 4. 页面怎么用

### 左侧：模板库

- 可以按分类查看模板
- 可以搜索关键词
- 可以点星号收藏常用模板

### 中间：参数填写区

- 选择模板后，会出现对应字段
- 把你的主题、背景、原文、目标等填进去
- 可以选择语气、长度和输出方式
- 还可以加“补充背景”和附加要求

### 右侧：生成结果

- 自动生成一段完整提示词
- 点“复制提示词”就能直接贴给 AI
- 点“保存本地”会把当前状态记到浏览器本地存储

## 6. 常见问题

### 页面打不开

先检查服务是否已经启动，再确认当前端口是否是 `18888`。

可以用下面的命令快速验证：

```powershell
curl http://localhost:18888/
```

如果返回 HTML 内容，说明服务是通的。

### 页面能打开，但没有内容

优先直接访问：

```text
http://localhost:18888/PromptWorkbench.html
```

### 修改模板后想重新发布

如果只是改本地文件，刷新浏览器即可。

如果要同步到 GitHub，执行正常的 `git add`、`git commit`、`git push` 流程即可。
